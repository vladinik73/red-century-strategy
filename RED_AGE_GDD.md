## I. Концепция игры

**Red Age** — пошаговая 4X grand strategy, вдохновлённая Civilization и Polytopia.

### Ключевые принципы

- Стратегическая глубина без избыточной сложности
- Асимметричные цивилизации
- Динамичная дипломатия и политика альянсов
- Несколько путей к победе
- Честная модель PvP
- Полная система реплеев и наблюдения

### Платформы

Кросс-платформенная (Mobile + PC)

### Игровые режимы

| Режим | Состав |
|---|---|
| PvE | 1 игрок + 6 ИИ |
| Live PvP | 2+ игроков + боты |
| Async PvP | 2+ игроков + боты |

## II. Режимы и параметры карты

### PvE карта

| Параметр | Значение |
|---|---|
| Размер | 80×80 |
| Континенты | 4–6 |
| Острова | 5–10 |
| Городов | 50–100 |
| Мин. расстояние между столицами | ≥10 клеток по оси |

### PvP карта

| Параметр | Значение |
|---|---|
| Размер | 50×50 |
| Континенты | 3–4 |
| Острова | 3–5 |
| Городов | 30–50 |
| Мин. расстояние между столицами | ≥7 клеток по оси |

### Города на карте

- Предгенерация при создании карты
- Минимум 2 клетки между городами
- Нельзя строить вручную
- Только захват

### Типы клеток

| Тип | Эффект |
|---|---|
| Равнина | Стандартная |
| Лес | −1 движение, +1 защита |
| Горы | +1 обзор, −1 движение, +1 урон при атаке с горы, −1 урон при атаке на гору с равнины; вход после Военной технологии ур. 2 |
| Вода | — |
| Река | −1 движение; мост = дорога |
| Пустыня | Низкая плотность ресурсов |

> Ресурсы видны сразу при открытии клетки.

|---|---|

## IV. Города

### Уровень города

- Лимит юнитов в городе = уровень города
- Производство: 1 юнит за ход
- Если город **свободен** (нет гарнизона) → производство мгновенное

### Повышение уровня

При повышении уровня игрок выбирает одно из:

1. **Расширение территории**
2. **Защитное укрепление** (+2 к защите города)
3. **Специализация** (+10% доход города)

### Защита города (уровни укреплений)

| Уровень | Бонус | Стоимость |
|---|---|---|
| 1 | +2 защита | 100 денег |
| 2 | +3 защита | 200 денег |

> Осада снижает бонус защиты на 1.

### Порты

- Строительство кораблей только в городе с портом
- Лимит кораблей на порт = уровень города
- При захвате города порт сохраняется

---

## V. Интеграция

После захвата города:

- **4 хода** интеграции
- Нет производства
- Нет магистрали
- Не учитывается в победных условиях
- Если возвращён прежнему владельцу в течение 4 ходов → штрафы отменяются

---

## VI. Магистраль

- Учитывается только сеть, связанная со **столицей**
- Существует только **одна** активная сеть
- Мост = дорога
- Disconnected city → теряет все бонусы сети

### Формула NetworkBonus

```
NetworkBonus% = (ConnectedCities × 2) + (TotalRoadTiles × 0.1) + (RoadLevelSum × 1)
Максимум: 40%
```

### Алгоритм пересчёта

Пересчёт происходит **немедленно** при:
- Строительстве дороги
- Разрушении дороги
- Захвате города

## VII. Боевая система

### Базовые параметры

| Параметр | Значение |
|---|---|
| Базовый HP | 10 |
| Базовый урон | 3 |
| Минимальный урон | 1 |
| Округление | вниз |

### Ветеранская система

После 3+ убийств → юнит переходит на следующий HP-тир.

### Серийный удар

- Если цель убита → возможна следующая атака
- Цепочка продолжается пока цель погибает
- После атаки без убийства — цепочка останавливается

### Осада

Если вражеский юнит соседствует с городом → −30% к производству.

### Окружение

- Учитываются 8 соседних клеток (включая диагонали)
- При 4+ врагах → −1 к защите
- Если один из окружения уничтожен — штраф снимается

### Горы

| Ситуация | Эффект |
|---|---|
| Атака с горы | +1 урон |
| Атака на гору с равнины | −1 урон |
| Обзор с горы | +1 |
| Вход на гору | Требует Военная технология ур. 2 |

### Результат боя

- Цель погибла → атакующий занимает клетку
- Цель выжила → атакующий остаётся на месте

---

## VIII. Авиация

- Базируется **только** в городе
- Количество авиации в городе: `floor(уровень города / 2)`
- Может атаковать наземные и морские цели
- Не может занимать клетки вне города
- Может быть атакована дальнобойными юнитами
- Дополнительных классов ПВО нет — перехват осуществляется стандартными дальнобойными юнитами

---

## IX. Морской бой и высадка

- Наземные юниты **не атакуют** корабли
- Корабли могут атаковать города с моря
- Высадка требует **порт**
- При разрушении порта морские юниты не могут базироваться

## X. Технологии

**3 ветки, 5 уровней каждая. Глобальные (без жёстких контр-веток).**

### Военная ветка

| Уровень | Разблокировка |
|---|---|
| 1 | Улучшение пехоты |
| 2 | Доступ к горам |
| 3 | Осадные технологии |
| 4 | Элитные войска |
| 5 | Военная доктрина |

### Экономическая ветка

| Уровень | Разблокировка |
|---|---|
| 1 | Дороги I |
| 2 | Порты |
| 3 | Дороги II |
| 4 | Магистральная логистика |
| 5 | Инфраструктурный максимум |

### Научная ветка

| Уровень | Разблокировка |
|---|---|
| 1 | Исследовательский центр |
| 2 | +10% науки |
| 3 | +20% науки |
| 4 | Ускорение технологий |
| 5 | Технологический прорыв (запускает таймер победы) |

## XI. Победные условия

### Приоритет при одновременном выполнении

**Военная > Экономическая > Технологическая**

Проверка всех условий — в конце полного раунда.

### Военная победа

| Режим | Условие |
|---|---|
| PvE | 60% интегрированных городов |
| PvP | 55% интегрированных городов |

### Экономическая победа

```
CityPower = сумма уровней интегрированных городов
Победа = контроль 75% суммарного CityPower карты (напрямую или через альянс)
Таймер удержания: 5 ходов подряд
При падении ниже 75% — таймер сбрасывается
```

### Технологическая победа

- После изучения Научной ветки ур. 5 → запускается таймер **10 ходов**
- Любой захват **интегрированного** города этой цивилизации → таймер сбрасывается
- Если город возвращён в течение 4 ходов интеграции → таймер **не** сбрасывается

### Победа альянса

- Если победное условие выполнено альянсом → победителем считается **лидер альянса** (игрок)
- Если лидер ИИ → игрок проигрывает

## XII. Стабильность

**Диапазон:** 0–100  
**Старт:** 70  
**Авто-восстановление:** +1 за ход

### Ускорение за деньги

| Трата | Прирост |
|---|---|
| 50 денег | +2 |
| 100 денег | +5 |

### Штрафы

| Событие | Изменение |
|---|---|
| Захват города | −2 |
| Потеря города | −4 |
| Потеря столицы | −8 |
| Разрыв союза | −2 |
| Оккупированный город (каждый) | −1 за ход |

### Пороговые эффекты

| Диапазон | Эффект |
|---|---|
| 80–100 | +5% доход, +1 ОД, +5% мораль (Процветание) |
| 60–79 | Норма |
| 40–59 | −5% доход |
| 20–39 | −10% доход, −1 ОД |
| 0–19 | −20% доход, −2 ОД, шанс восстания |

### Восстание города

При стабильности 0 и успешной проверке (10% за ход):
- Город становится **нейтральным**
- Уровень города сохраняется
- Гарнизон отсутствует
- Требуется повторный захват

---

## XIII. Мораль

```
Мораль = 100 при стабильности ≥ 70
Каждые 10 пунктов ниже 70 → −2 мораль
При стабильности 0 → шанс восстания 10% за ход
```

### Влияние морали на урон

| Стабильность | Модификатор урона |
|---|---|
| 80–100 | +10% |
| 60–79 | норма |
| 20–39 | −10% |
| 0–19 | −20% |

---

## XIV. Дипломатия

### Состояния

- Война
- Нейтралитет
- Альянс

### Альянс

- Максимум **2 союза** одновременно
- Нельзя разорвать раньше **6 ходов**
- После разрыва — **3 хода cooldown**
- Бонусы: +5% доход, +5% производство, +5% мораль (макс. +10% от стопки)
- Победа альянса: все участники побеждают

### Диверсии и война

Использование киберподразделения на территории **нейтральной** цивилизации → автоматическое объявление войны.

## XV. Очки действия — полная модель

```
ОД = 5 (база)
    + floor(Сумма уровней интегрированных городов / 5)
    + floor(Количество технологий / 3)
    + StabilityModifier
    + NetworkModifier
    + AllianceModifier
    - OccupationPenalty
    - CyberPenalty
```

### Модификаторы

| Модификатор | Формула |
|---|---|
| StabilityModifier | 80–100: +1; 20–39: −1; 0–19: −2 |
| NetworkModifier | floor(NetworkBonus% / 10) |
| AllianceModifier | +1 если активен союз |
| OccupationPenalty | −1 за каждые 2 неинтегрированных города |
| CyberPenalty | −1 при активном «Сбое» |

> **ASSUMPTION:** Минимальное значение ОД за ход = 1 (нижний порог не указан в документе, рекомендуется установить).

## XXII. Формула боя

```
Damage = BaseDamage + TerrainBonus + MoraleModifier - DefenseModifier

BaseDamage      = параметр юнита
TerrainBonus    = +1 (атака с горы) / −1 (атака на гору с равнины)
MoraleModifier  = +10% (стаб. 80–100) / −10% (20–39) / −20% (0–19)
DefenseModifier = +1 (лес) / +2 (укреплённый город) / −1 (окружение 4+)

Минимальный урон = 1
Округление: вниз
```

## XXV. ИИ — система и математическая модель

### Уровни сложности

| Параметр | Easy | Normal | Hard | God |
|---|---|---|---|---|
| Экономика | ×0.85 | ×1.0 | ×1.10 | ×1.10 |
| ОД | ×0.85 | ×1.0 | ×1.10 | ×1.0 |
| Мораль | −10% | норма | +5% | +5% |
| Использование ОД | 75% | 92% | 98% | 100% |
| Порог атаки | 70 | 55 | 45 | 35 (30 при стратег. цели) |
| Макс. атак за ход | 3 | 5 | 7 | 9 |

> Все боты в матче имеют **одинаковую** сложность.  
> ИИ **не имеет** чит-видимости.

### Математическая модель

```
LeaderIndex = 0.4 × Military + 0.35 × Economy + 0.25 × Progress

Military = 0.6 × (Доля интегрированных городов) + 0.4 × (Сила армии)

Economy = 0.7 × (Доля суммарных уровней городов) + 0.3 × (NetworkBonus%)

Progress = 0.5 × (Уровень науч. ветки / 5 × 100)
         + 0.3 × (Общий уровень технологий / 15 × 100)
         + 0.2 × (Прогресс тех-таймера)

ThreatScore = LeaderIndex + ProximityBonus

EnemyScore = 0.45 × ThreatScore
           + 0.25 × ProximityScore
           + 0.20 × OpportunityScore
           + 0.10 × GrievanceScore

CityPriority = 30 × CapitalFlag
             + 20 × NetworkNodeValue
             + 20 × VictoryDenialValue
             + 15 × WeakDefenseValue
             + 15 × StrategicPositionValue

WarScore = 0.55 × OpportunityScore
         + 0.25 × max(0, ThreatScore − 50)
         + 0.20 × Revenge

AllyScore = 0.45 × BenefitScore
          + 0.25 × RelationScore
          + 0.30 × AntiLeaderNeed

BreakScore = 0.5 × BetrayalGain
           + 0.3 × ThreatScore
           + 0.2 × OpportunityElsewhere
```

## XXVI. PvP система

### Live PvP

- Последовательные ходы
- Таймер хода:

```
Время = 90 сек (база) + (2 × кол-во юнитов) + (3 × кол-во городов)
Максимум: 180 сек
```

### Async PvP

- Пропуск хода = ход завершён
- ИИ не заменяет игрока
- Рейтинг сохраняется при пропуске

### Выход игрока (Live PvP)

- Покидает матч → поражение
- Цивилизация переходит под ИИ
- Партия продолжается

## XXXI. Дополнительные уточнения баланса

### Киберподразделение

- Нельзя применять «Сбой» на столицу
- Нельзя разрушать дороги в радиусе 1 клетки от столицы
- Нельзя разрушать порт столицы

### Авиация

- Дополнительных классов ПВО нет
- Перехват — стандартными дальнобойными юнитами

### Рой автономных дронов

- Бонус от группировки: максимум **+1**, независимо от числа соседних роёв

### Гиперзвуковой ракетный комплекс

- Не может атаковать столицу
- Максимум **1 удар за ход** на цивилизацию

### Восстание города

- 0 стабильности + проверка 10% → город становится нейтральным
- Уровень сохраняется, гарнизон отсутствует, нужен повторный захват

## XXXII. UI/UX PvE Web (Minimalistic Mode)

### Общий стиль

- Минималистичный интерфейс (вдохновлён Polytopia)
- Светлая цветовая схема
- Реалистичная, упрощённая карта
- Нет границ стран — только границы городов

### Layout

```
┌────────────────────────────────────────────────┐
│              Верхняя панель ресурсов            │ ← фиксированная
├──────────┬─────────────────────────┬────────────┤
│  Левая   │                         │   Правая   │
│  панель  │      Центральная        │   панель   │
│ "Империя"│         карта           │ (контекст) │
│ (скрыв.) │                         │  (скрыв.)  │
├──────────┴─────────────────────────┴────────────┤
│       Нижняя панель: лог / технологии / дипл.   │
└────────────────────────────────────────────────┘
```

- Левая и правая панели сворачиваются
- При выборе объекта правая панель автоматически раскрывается

### Карта

- Плоские цвета без мелких текстур
- Территории принадлежат городам
- Границы города — тонкий цветной контур
- **Overlay режимы:** Сеть / Ресурсы / Военная угроза / Интеграция

### Поведение юнита

| Действие | Поведение |
|---|---|
| Выбор юнита | Подсвечиваются только клетки движения |
| Перемещение | Мгновенное, без анимации |
| Отмена перемещения | Невозможна |
| Враг в радиусе атаки после хода | Юнит остаётся активным |
| Атака | Клик по врагу |
| Кнопки "Атаковать" / "Переместить" | Отсутствуют |

### Панель юнита

Отображает:
- Название юнита
- Текущий HP (только цифра; если ≤30% от макс. — цифра **красная**)
- Кнопки: **Вылечить**, **Расформировать**

### Кнопка «Вылечить»

| Параметр | Значение |
|---|---|
| Стоимость | 1 ОД |
| Эффект | +3 HP |
| Ограничение | Нельзя, если рядом враг |

### Кнопка «Расформировать»

- Юнит уничтожается
- Возвращает **50% стоимости** в деньги

### Атака

- Разрешается мгновенно
- Всплывающие числа урона
- Отмена невозможна
- После атаки юнит становится **неактивным**

### Серийный юнит

- После убийства цели остаётся **активным**
- Может атаковать следующую цель
- Цепочка до момента, когда цель не погибает

### Визуальные состояния юнита

| Состояние | Индикатор |
|---|---|
| Активный | Тонкий светлый контур |
| Неактивный | Контур отсутствует |
