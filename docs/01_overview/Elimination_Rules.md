# Elimination Rules (v4.23) — Canon MVP

Канонические правила элиминации цивилизации и связанные эффекты.

## Elimination condition

Цивилизация **элиминирована**, если у неё **0 городов**.

- Наличие юнитов **не важно** — условие проверяется только по городам.
- Проверка выполняется в конце хода цивилизации (см. `docs/01_overview/Turn_Pipeline.md`).

## On elimination

При элиминации цивилизации:

- **Все юниты** элиминированной цивилизации **немедленно удаляются** с карты.
- Юниты не переходят нейтральными и не остаются на карте.

## Turn order

- Элиминированная цивилизация **пропускается** в порядке хода.
- Элиминированная цивилизация **не получает ход** (PHASE 0–4 не выполняются для неё).

## Diplomacy / history

- Дипломатические отношения могут сохраняться как **исторические данные** для replay и аналитики.
- В матч-логике элиминированная цивилизация считается **неактивной** (не участвует в расчётах, не получает ход).

## Victory checks

- После элиминации **сразу пересчитывается** прогресс побед:
  - городские доли (военная победа),
  - CityPower (экономическая победа)
- Расчёт ведётся по **актуальному набору активных цивилизаций** (элиминированные не учитываются).

## UI/UX — Defeat

- При поражении (элиминации) **игрок** видит экран **Defeat**.
- End-match flow: см. `docs/01_overview/MVP_Player_Journey.md` (цепочка экранов: Victory/Defeat → Rematch / Exit).
- Связь с основным экраном: `docs/10_uiux/Main_Game_Screen.md`.
