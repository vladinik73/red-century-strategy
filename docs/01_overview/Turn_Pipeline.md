# Turn Pipeline (Canonical) — v4.28

Этот документ задаёт строгий порядок фаз хода (turn order) для Red Age и является каноном для реализации core loop.

## Принципы

- Игра пошаговая, ходы **последовательные**.
- Один «раунд» = ходы всех цивилизаций, но **вся логика** (таймеры/доход/победа) считается **в конце каждого хода**.
- Все вычисления детерминированы.
- `Undo` отсутствует.

---

## Структура хода цивилизации

### PHASE 0 — Start of Turn (System)

Выполняется автоматически.

#### 0.1 Tick timers

**Города**
- `IntegrationTimer--` (если город в интеграции)
- `DisruptionTimer--` (если на городе активен «Сбой»)
- `SiegeState` (если город под осадой — состояние активно, таймер если применяется)

**Тайлы**
- `DamagedRoadTimer--` (для тайлов дорог со статусом `DamagedRoad`)

#### 0.2 Expired status cleanup

Если таймер стал `0`:

- Интеграция: город становится **интегрированным**, снимаются оккупационные дебаффы.
- Disruption: снимаются эффекты «Сбоя» (доход/ОД).
- DamagedRoad: дорога считается восстановленной (тайл снова учитывается сетью).
- Осада: если осада снята (условие/таймер) — снимается штраф осады.

#### 0.3 Reset per-turn counters

- `BoostScienceThisTurn = 0`
- `StabilityBoostUsedThisTurn = false`
- `HasActedThisTurn = false` для всех живых юнитов этой цивилизации

#### 0.4 Rebellion check (Stability = 0)

Для каждого города игрока:

- Если `Stability == 0`:
  - `10%` шанс → город становится **нейтральным** (уровень сохраняется, гарнизона нет, нужно захватывать заново).

Это происходит **до** начисления дохода за ход.

---

### PHASE 1 — Income Phase (System)

#### 1.1 Calculate MoneyPerTurn

Деньги начисляются согласно канонической модели денег (`Money_Model.md`) с учётом множителей:
- `StabilityMultiplier`
- `AllianceMultiplier`
- `SiegeMultiplier`
- `CyberIncomeMultiplier` (если применимо)
- `NetworkBonus` (если применимо)

#### 1.2 Apply Money

- `Player.money += MoneyPerTurn`

#### 1.3 Calculate SciencePerTurn

Наука начисляется согласно канонической модели науки (`Tech_Progression.md`), включая модификаторы (например, осада снижает CityBonus согласно канону).

#### 1.4 Apply Science

- `Player.science += SciencePerTurn`

#### 1.5 Fix Boost Cap

- `SciencePerTurn_ForBoostCap = SciencePerTurn` (значение до бустов, используется для лимита Boost Science в PHASE 3).

---

### PHASE 2 — Action Points (AP) calculation (System)

ОД рассчитываются согласно `Action_Points.md`:

- `Player.currentAP = max(1, AP)`
(минимум `1` ОД за ход)

---

### PHASE 3 — Action Phase (Player)

Игрок выполняет действия и тратит ОД:

- Перемещение юнитов (мгновенно)
- Атака (клик по врагу при выбранном юните)
- Производство юнитов (по правилам `Production_Rules.md`)
- Строительство (дороги/порты/мосты — по канону)
- Чинка дорог (`DamagedRoad`) — по канону
- Кибердействия — по канону
- **Boost Science** (1 ОД) — конвертирует деньги в науку; лимит за ход: `BoostScienceThisTurn ≤ floor(SciencePerTurn_ForBoostCap × 2)` (см. `Tech_Progression.md`)
- **Boost Stability** (1 ОД + деньги, раз в ход на цивилизацию):
  - cost: `1 AP` + деньги (`50` → +2 Stability или `100` → +5 Stability)
  - requires: ≥1 интегрированный город у цивилизации
  - sets: `StabilityBoostUsedThisTurn = true`
- Прочие действия юнитов (`Heal`, `Disband`) — по канону UI/UX

Правила:
- ОД списываются **немедленно**.
- Отмена действий **не предусмотрена**.
- **1 действие на юнит за ход:** каждый юнит может выполнить только одно действие (Move / Attack / Heal / Disband / Special). После выполнения юнит помечается `HasActedThisTurn = true` и становится недоступен до следующего хода цивилизации.

---

### PHASE 4 — End of Turn (System)

#### 4.0 Elimination check

Если у цивилизации **0 городов** → цивилизация **элиминирована**:
- Удалить цивилизацию из turn order (пропуск в порядке хода).
- Эмитировать событие **ELIMINATION** (`{ civ_id }`).
- Удалить все юниты элиминированной цивилизации.

Проверка выполняется **до** обновления победных таймеров и проверки победы. Канон: `docs/01_overview/Elimination_Rules.md`.

#### 4.1 Update victory timers

Проверяются условия побед (канон `Victory_Rules.md`):

- Военная победа (≥60% интегрированных городов)
- Экономическая победа (≥75% глобального CityPower **только своими** интегрированными городами), удержание 5 ходов
- Технологическая победа (Science L5), удержание 10 ходов, сброс таймера при захвате города этой цивилизации (по канону)

Если условие выполняется — соответствующий таймер увеличивается на 1.
Если условие нарушено — таймер сбрасывается.

#### 4.2 Victory check

Если таймер достиг порога:
- Military: сразу (или 1 ход, если так задано в каноне побед)
- Economic: `>= 5`
- Tech: `>= 10`

→ матч завершается.

#### 4.3 End turn

Переход хода к следующей цивилизации.

---

## End of Round

Отдельной «фазы конца раунда» нет. Все проверки делаются в конце хода каждой цивилизации.

---

## Инварианты порядка

- Таймеры и очистка статусов тикают **до** дохода.
- Бунт проверяется **до** дохода.
- Доход (Money/Science) начисляется **до** расчёта ОД.
- Победа проверяется **в конце каждого хода** (не раунда).
